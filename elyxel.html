The Idea Vending Machine

Learnings from One Year of Building an Open Source Elixir Application

If you're ever in Japan, one of the many things that might stand out are the multitude of glowing candy colored mechanical cubes lining the streets. These machines are often filled with a dizzying array of surprisingly tasty snacks and beverages. It's hard to resist trading the extra yen in your pocket for one of these delectable treats.

I can't help but relate the idea of working on a project to the concept of these vending machines. You trade time and effort to bring something into reality. Beyond the seductive temptation of ideas calling to you from behind the glass window, it's difficult to prove the merit of an idea without first investing time and work.

I traded my free time last year to learn about and build an Elixir application. The result, Elyxel is a basic community tool much like Hacker News designed to be a space to gather and share stories. My goals were simple; finish and learn new skills along the way. The following is a detailed account of the things I learned along the way.

Additionally, Elyxel will be open sourced in the hope that it might be beneficial to anyone trying to learn from what I did. This piece is geared towards someone who is familiar with the concepts of programming and general web development.

Avoiding Design Limbo

I wanted to start with a fairly unoriginal simple concept that was familiar to me. At the time, solving the well defined problem of community software was particularly appealing. I was getting fatigued from the constant negative rhetoric that seemed to dominate the conversation on forums I frequent. That coupled with a desire to learn new technology was all the kindling I needed to get started.

The actual design of the site took a minimal amount of time. I opted for a simple relatively sparse interface. In the past I would spend a lot of time up front on the design phase of the project. It's easy (and honestly quite fun) to get lost in the design process.

Elixir & Phoenix

To gain better leverage, I decided to use this spare time as an opportunity to learn a new programming language and framework. Elixir and Phoenix came out on top after some researchâ€”it was fast, being actively developed and most importantly easy to get started with.

After settling on my choice, I knew I needed a server to host the finished application. Instead of using an automated service like Heroku I set about learning how to provision and setup a small virtual private server (VPS). Now having gone through the process, I've gained a greater appreciation and understanding of the setup infrastructure. I highly recommend doing it at least once.

If you're interested in doing the same, I've listed the major steps of the process below. The entire process took about a month of learning and understanding best practices. I had to cut some of these explorations short as each thread I pulled on unearthed a hundred more. There's a staggering amount of low level tech built by brilliant folks we rely on everyday.

  1. Installed Ubuntu
  2. Setup password less login via SSH
  3. Installed NGINX
  4. Installed htop for monitoring
  5. Pointed the elyxel.com domain to the server
  6. Setup SSL via Let's Encrypt

Note: Tutorials for this stuff are pretty easy to find. In fact most of the basics can be found on in one place: Digital Ocean's site.

The Meat & Potatoes

The tutorial on the phoenix site was really helpful but like any nascent technology it is not fairly exhaustive and assumes some familiarity with the process of building a modern web application.

There was an upfront cost of having to learn elixir before I could build anything. While the syntax was familiar there were a few new concepts to grapple with due to elixir being a functional programming language. There are many great tutorials to get you started here.

Phoenix required a similar learning curve, but it had enough similarity to rails that this was a relatively smooth process as well.

Once I had the basics it was a matter of building things piece by piece. I broke it down into several smaller parts based on features that have to be present in a 1.0.

Interesting bits of code

  • pagination
  • voting algorithm
  • pagination
  • markdown support in comments

1.0 build requirements: - Written brief of what I'm trying to achieve - Sketch & Wireframes - Server setup and config - Landing Page - Signup & Invite system flow - App features: - Top stories page - Recent stories page - Submit page - Comments - Profiles - Voting system

The made challenges I encountered during the build process was the lack of robust tutorials and best practices. In hind sight this was actually favorable because it ended up being a rewarding challenge to figure out things on my own. I wouldn't recommend this process if you're under a deadline, but if the goal is learning then it definitely is valuable.

I also tried to pick simpler libraries to utilize. The goal here was to be able to understand the tools I was using and avoid cruft. We've all heard the horror stories of including some mega complex library to achieve a relatively simple goal.

The login and invite system was particularly tricky even though I used a simple library called open maize. It was great but I found myself getting really paranoid about missing some big security feature and leaving myself exposed.

One huge source of inspiration here is the open source community Lobster's built in rails. The code base was well built and accessible. I used some of the implementations I discovered from here. For example this particularly elegant bit of code allowing for human readable timestamps. Cool stuff. This serves as a natural segue into why I'm open sourcing Elyxel.

I decided to keep the code closed partially due to the worry of exposing really dumb mistakes or sloppy code. But I learnt a lot from a lot of the tutorials that I discovered along the way and I feel like this might be a good way to give back and also open the floor to improvements.

So people can learn from my mistakes and hopefully improve.

Fatigue

One thing that stuck with me throughout this project was how challenging it was to work a full work day and spend a few extra hours at night. Particularly because I was doing really interesting rewarding work as well. There were so many times where I was too exhausted to come home and work on Elyxel even though I was super excited to. This was deeply frustrating and I'm not sure what the solution here is. I just felt I had to share this. I always admire folks who are able to do both consistently well.

One good mantra here to keep moving as well was not letting perfection get in the way of something being good. There is endless amount of ways to get something better but I still have to start feeling comfortable sharing things that might not quite be where I think it should be.

Race to 1.0

I ended up finishing my 1.0 in January but have not done any work on it since. This is partially due to a massive project at work, moving and generally wanting some distance from the project.

What's next?

Truthfully, I'm honestly not sure. Now that it's out I'm not quite sure what to do next. 1 option is to push and try and build a real community out of Elyxel, which is a challenge of a different kind. If you've read through most of this, first of all wow, and secondly you might be interesting in joining. Send me an e-mail if you're interested.
